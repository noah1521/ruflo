### Scoring Model Adjustment — Zero Confirmed FAILs but Data Gaps Present

The current binary PASS/FAIL system breaks down when data is unavailable. "No confirmed fail" is not "confirmed pass." The difference is the confidence interval around each criterion.

**Proposed: Confidence-Weighted Pass (CWP) scoring**

| Criterion State | Score | Meaning |
|----------------|-------|---------|
| Confirmed PASS (live data) | 1.0 | Green. Trade-eligible. |
| Unverified / No data available | 0.5 | Yellow. Adds uncertainty, not confidence. |
| Confirmed FAIL | 0.0 | Red. Automatic disqualification. |

Rules:
- Sum criterion scores across all 5 criteria. Maximum = 5.0.
- Setup must score 4.0 or above to be trade-eligible.
- A single confirmed FAIL disqualifies the setup regardless of composite score.
- An unverified criterion scores 0.5 — not 1.0.
